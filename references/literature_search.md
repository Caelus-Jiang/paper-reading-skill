# 文献检索规则

## 目录

- 来源优先级
- 检索策略
- 证据分级
- 可选社区入口
- 报告规则

## 来源优先级

按以下顺序选择可核查来源：

1. 会议、期刊或出版社官方页面
2. OpenReview 等正式评审页面
3. arXiv 原文
4. 作者或机构项目页、官方代码仓库
5. 可核验的机构镜像

明确区分 peer-reviewed 论文和 preprint。聚合摘要、营销稿和博客不能支撑学术结论。

## 检索策略

每次围绕一个技术问题检索：

- 方法族 + limitation / failure mode
- 关键假设 + challenge / counterexample
- 任务 + benchmark + comparison
- 标题或作者 + discussion / response

至少覆盖支持路径与竞争/反对路径。只引用已经打开并核对原文、摘要或正式页面的工作。

## 证据分级

- **直接支持**：研究问题、设置和指标与目标主张直接对应。
- **间接支持**：只支持机制的一部分或相邻任务。
- **竞争解释**：用更简单或不同机制解释类似结果。
- **未形成证据**：主题相关，但不能支持或反驳目标主张。

不要把主题相似自动写成支持证据。

## 可选社区入口

知乎、CSDN 等社区文章只可作为额外阅读入口：

- 不计入学术证据。
- 不要求固定数量。
- 不得因为缺少社区文章而阻塞报告。
- 只有能明确对应目标论文、具有作者署名和实质分析时才列出。

## 报告规则

- 在 4.5 相关论文表中给出真实标题、基础 arXiv ID、作者/年份、来源/类型和关系说明，但不添加网络链接。
- 将正文实际使用文献的外部核查链接统一写入附录 B。
- preprint 明确标注。
- 只总结已经确认的结论。
- 未找到直接证据时写“当前未检索到明确支持或反对证据”。
- 附录 B 只列正文实际使用的外部文献。
